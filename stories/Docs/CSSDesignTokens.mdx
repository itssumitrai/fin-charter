# CSS Design Tokens

fin-charter resolves chart and series colors through a three-tier priority system:

1. **Explicit JS options** — values passed directly to `createChart()` or `chart.applyOptions()` always win.
2. **CSS variables** — `--fc-*` custom properties read from the container element via `readCSSTheme()`.
3. **Built-in defaults** — the library's own fallback values, used when neither JS options nor CSS variables are set.

This means you can theme the entire chart purely via CSS without writing any JS configuration. Define `--fc-*` variables on the chart container (or any ancestor), call `readCSSTheme()`, and spread the result into `createChart()`.

---

## Usage Example

**Define variables in CSS:**

```css
.my-theme {
  --fc-candle-body-up: #26a69a;
  --fc-candle-body-down: #ef5350;
  --fc-candle-wick-up: #26a69a;
  --fc-candle-wick-down: #ef5350;
  --fc-bg: #1e1e2e;
  --fc-text: #d1d4dc;
}
```

**Apply the theme and create the chart:**

```ts
import { readCSSTheme, createChart } from '@itssumitrai/fin-charter';

const container = document.getElementById('chart')!;
container.classList.add('my-theme');

const { chartOptions, seriesDefaults } = readCSSTheme(container);
const chart = createChart(container, { autoSize: true, ...chartOptions });

// Series will pick up candle colors from CSS automatically
const series = chart.addSeries({ type: 'candlestick' });
```

Only variables that are defined in CSS are included in the result. Undefined variables are omitted so they do not override library defaults.

## Backward Compatibility

The legacy aliases `--fc-candle-up` and `--fc-candle-down` continue to work and map to the candle body colors. The more specific variables take priority:

| Legacy variable | Maps to |
|---|---|
| `--fc-candle-up` | `--fc-candle-body-up` (fallback only — more specific variable wins) |
| `--fc-candle-down` | `--fc-candle-body-down` (fallback only — more specific variable wins) |

If both `--fc-candle-body-up` and `--fc-candle-up` are defined, `--fc-candle-body-up` takes priority.

## Refreshing After CSS Changes

If you update CSS variables at runtime (e.g., switching a class), call `chart.refreshCSSTheme()` to re-read all `--fc-*` variables from the container and trigger a repaint:

```ts
// Switch theme class
container.classList.remove('my-dark-theme');
container.classList.add('my-light-theme');

// Re-read and repaint
chart.refreshCSSTheme();
```
